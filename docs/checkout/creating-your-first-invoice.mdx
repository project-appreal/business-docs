# Creating Your First Invoice

This guide walks you through creating an invoice using the Business API.
When the invoice is created, you receive a `paymentUrl` that you can redirect your customer to.

## Create an invoice

Send a `POST` request to the invoices endpoint with the payment details:

<Tabs>
  <TabItem value="curl" label="cURL">

```bash
curl -X POST https://api.appreal.com/api/business/v1/invoices \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "customerEmail": "customer@example.com",
    "invoice": {
      "amount": 100,
      "destinationCurrency": "USDT"
    },
    "redirectUrl": "https://yourstore.com/order/complete",
    "webhookUrl": "https://yourstore.com/api/webhooks/appreal",
    "reference": "order-12345"
  }'
```

  </TabItem>
  <TabItem value="js" label="Node.js">

```js
const response = await fetch('https://api.appreal.com/api/business/v1/invoices', {
  method: 'POST',
  headers: {
    'x-api-key': process.env.API_KEY,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    customerEmail: 'customer@example.com',
    invoice: {
      amount: 100,
      destinationCurrency: 'USDT',
    },
    redirectUrl: 'https://yourstore.com/order/complete',
    webhookUrl: 'https://yourstore.com/api/webhooks/appreal',
    reference: 'order-12345',
  }),
});

const data = await response.json();
console.log(data.paymentUrl); // Redirect customer here
```

## Request body

| Field | Type | Required | Description |
|---|---|---|---|
| `customerEmail` | string | Yes | Customer's email address |
| `invoice.amount` | number | Yes | Payment amount in the destination currency |
| `invoice.destinationCurrency` | string | Yes | Currency to settle in (e.g., `USDT`, `USDC`) |
| `redirectUrl` | string | No | URL to redirect the customer after payment |
| `webhookUrl` | string | No | URL to receive status change notifications |
| `reference` | string | No | Your internal order reference |
| `nonce` | string | No | Unique identifier for idempotency |

## Response

A successful request returns the created invoice:

```json
{
  "id": "inv_abc123def456",
  "number": "INV-2026-001",
  "status": "PREPARED",
  "paymentStatus": "UNPAID",
  "destinationCurrency": "USDT",
  "amount": "100.00",
  "paymentUrl": "https://pay.appreal.com/invoice/inv_abc123def456",
  "expiredAt": "2026-04-14T10:15:00.000Z",
  "createdAt": "2026-04-14T10:00:00.000Z"
}
```

| Field | Description |
|---|---|
| `id` | Unique invoice identifier |
| `number` | Human-readable invoice number |
| `status` | Current invoice status (`PREPARED`) |
| `paymentStatus` | Current payment status (`UNPAID`) |
| `paymentUrl` | Hosted checkout page URL â€” redirect your customer here |
| `expiredAt` | When the invoice expires if no payment method is selected |

## What happens next

1. **Redirect the customer** to the `paymentUrl`
2. The customer selects a cryptocurrency and network on the payment page
3. The invoice status changes from `PREPARED` to `ACTIVE`
4. Once the customer sends payment, the status moves to `CONFIRMING`
5. After blockchain confirmations, the status becomes `COMPLETED`
6. You receive a webhook at each status change
